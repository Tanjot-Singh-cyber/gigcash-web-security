GIGCASH ROADMAP OVERVIEW

Goal:
Become strong in Web Application Security with proof-based learning
leading to gigs, internships, and product roles.

Philosophy:
- Reasoning-first
- Backend understanding before exploitation
- Minimal tools, maximum thinking

High-level phases:
1. Core Web Vulnerabilities
   - XSS (reflected, stored, DOM)
   - SQL Injection (including blind)
   - Logic flaws

2. Context & Execution Mastery
   - HTML, attribute, JS, URL contexts
   - Browser parsing behavior

3. Chaining & Realistic Scenarios
   - Multiple vulnerabilities together
   - Real-world style labs

4. Proof & Monetization
   - Clean GitHub writeups
   - Public proof of skill
   - Gigs / internships

Rules are defined in RULES.md.
Progress is tracked in progress/.
